The Problem

Manim is powerful
but inaccessible

The Manim Python library (used by 3Blue1Brown) creates stunning math animations — but demands knowledge of hundreds of class names, method signatures, and rendering conventions.

Educators and researchers with mathematical expertise but limited coding fluency are locked out.

LLM Shortcomings

  • Hallucinate non-existent class names
  • Invoke deprecated API methods
  • Produce scenes that render incorrectly
  • Static training — can't track API changes

Root Cause

Manim's API evolves rapidly. LLM weights cannot track every method rename or deprecation. The gap between training snapshots and the current API is the core failure mode.

3 Principal Contributions
1

Corpus Indexing

AST-based Python extraction, structured metadata, and overlapping chunking to produce a retrieval corpus that is semantically coherent and version-aware.

2

Self-Healing Loop

Feeds Manim error tracebacks back into the LLM with RAG context, enabling autonomous multi-round repair without human intervention (up to 3 rounds).

3

Logging & Analysis

Structured session logging enabling quantitative measurement of RAG benefit, error taxonomy, and self-healing effectiveness across all sessions.

Chunking Strategy

~800 token overlapping chunks. Chunks under 100 characters discarded as content-poor stubs. Each chunk stored with source type + file location metadata.

Self-Healing Loop — up to 3 autonomous rounds

On failure, the runtime error traceback + original RAG context are re-injected into the LLM for repair. A clean_code() post-processor always strips Markdown fences — a common LLM formatting failure. All calls routed through OpenRouter for model-agnostic substitution.

Success Rate

We benchmarked AutoManim across 100 complex mathematical prompts. The combination of RAG and Self-Healing nearly doubles the reliability of the system.

Future Horizons

Current Limitations

  • Dependency Chain: Requires LaTeX and FFmpeg environments properly configured.
  • Token Cost: Large RAG context increases per-prompt cost.
  • Multi-Scene Logic: Complex multi-file animations still require human oversight.

The Next Step

  • Multimodal Ingestion: Generating animations from whiteboard images or photos.
  • Manim-Specific SLM: Fine-tuning small language models specifically on the Manim AST.
  • Real-time Co-pilot: An IDE extension for live mathematical visualization scripting.

"Bridging the gap between conceptual mathematics and cinematic visualization."

Conclusion
RAG + Repair
= Reliability

AutoManim demonstrates that grounding LLM generation in real documentation and adding runtime self-repair loops significantly reduces hallucination and increases successful Manim animation rendering. Released as open-source to invite community extension.
